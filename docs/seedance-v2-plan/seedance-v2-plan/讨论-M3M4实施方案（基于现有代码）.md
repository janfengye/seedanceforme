## 二、四角色讨论

### 2.1 产品经理

**核心需求不变**：项目 → 集 → 镜头 → 版本，文件命名 `{代号}-{集}-{镜头}-{版本}-{昵称}.mp4`。

**基于现有代码的调整**：

1. **项目创建权限收紧**：当前所有用户都能创建项目，改为仅管理员。但现有项目数据不受影响——它们仍然可以被使用，只是普通用户不能再创建新项目。

2. **项目代号（code）是可选的**：已有项目不强制设置代号。只有设置了代号的项目，其下的镜头才能生成规范文件名。

3. **preferred_model 的交互**：在镜头编辑弹窗中增加"推荐模型"下拉。选择镜头生成视频时，自动预选该模型，用户可覆盖。这是一个"便捷默认值"，不是强制。

4. **版本号预览**：生成页面选择镜头后，显示"预计版本号：C"和"文件名预览：SN-1-1-C-ZS.mp4"。但实际版本号以后端分配为准（并发安全）。

5. **分期策略保持不变**：
   - 单任务和批量生成同时支持镜头关联
   - 镜头关联可选，不破坏"快速生成"体验

#### 新建服务文件

**`server/services/versionService.js`**：
```javascript
import { getDatabase } from '../database/index.js';

export function numberToVersionLabel(n) {
  let label = '';
  let num = n;
  do {
    label = String.fromCharCode(65 + (num % 26)) + label;
    num = Math.floor(num / 26) - 1;
  } while (num >= 0);
  return label;
}

/**
 * 为已完成的 output 任务分配版本号
 * 调用时机：updateTaskStatus 中 status='done' 且 shot_id 不为空时
 * 并发安全：better-sqlite3 同步 + Node.js 单线程
 */
export function assignVersionLabel(taskId) {
  const db = getDatabase();
  const task = db.prepare('SELECT id, shot_id, user_id FROM tasks WHERE id = ?').get(taskId);
  if (!task || !task.shot_id) return null;

  const count = db.prepare(
    'SELECT COUNT(*) as cnt FROM tasks WHERE shot_id = ? AND user_id = ? AND version_label IS NOT NULL'
  ).get(task.shot_id, task.user_id).cnt;

  const label = numberToVersionLabel(count);
  db.prepare('UPDATE tasks SET version_label = ? WHERE id = ?').run(label, taskId);
  return label;
}
```

**`server/services/filenameService.js`**：
```javascript
import { getDatabase } from '../database/index.js';

function sanitizeForFilename(str) {
  if (!str) return '';
  return str.replace(/[^\w\u4e00-\u9fff-]/g, '_').replace(/_+/g, '_').replace(/^_|_$/g, '');
}

export function buildStandardFilename(task) {
  const { project_code, episode_number, shot_number, version_label, nickname, username } = task;
  if (!project_code || episode_number == null || shot_number == null || !version_label) return null;

  // 优先用 nickname，其次用 username
  const displayName = nickname || username || null;
  if (!displayName) return null;

  const safeName = sanitizeForFilename(displayName);
  if (!safeName) return null;

  return `${project_code}-${episode_number}-${shot_number}-${version_label}-${safeName}.mp4`;
}

export function buildFallbackFilename(taskId) {
  return `video_${taskId}_${Date.now()}.mp4`;
}

export const FILENAME_QUERY = `
  SELECT
    t.id,
    t.version_label,
    p.code AS project_code,
    e.episode_number,
    s.shot_number,
    u.nickname,
    u.username
  FROM tasks t
  LEFT JOIN shots s ON t.shot_id = s.id
  LEFT JOIN episodes e ON s.episode_id = e.id
  LEFT JOIN projects p ON e.project_id = p.id
  LEFT JOIN users u ON t.user_id = u.id
  WHERE t.id = ?
`;
```

### 2.4 架构师

**整体方案审查**：

#### 与现有代码的融合点

1. **projects.user_id 的语义变化**：
   - 现在：表示"项目属于哪个用户"
   - M3 后：项目由管理员创建，`user_id` 变为"创建者 ID"。但所有登录用户都能看到项目（因为项目是团队共享的）。
   - **不需要修改 `user_id` 字段**——现有的 `GET /api/projects` 中 admin 已经能看到所有项目。当项目改为 admin 专属创建后，`user_id` 自然是管理员的 ID，普通用户通过 `isAdmin=false` 查询会看不到。

   **问题**：普通用户需要看到管理员创建的项目（用于在生成页选择镜头关联），但当前代码中 `isAdmin=false` 会过滤 `WHERE p.user_id = ?`。

   **解决方案**：修改 `projectService.getAllProjects`：
   ```javascript
   // 所有用户都能查看所有项目（项目是团队共享资源）
   // 但非管理员不能创建/编辑/删除（由路由中间件控制）
   export function getAllProjects() {
     const db = getDatabase();
     return db.prepare(`
       SELECT p.*, ... FROM projects p ORDER BY p.updated_at DESC
     `).all();
   }
   ```

   这是一个**语义性变更**——项目从"个人所有"变为"团队共享"。需要同时更新 `getProjectById` 去掉 user_id 过滤。

2. **draft → output 展开时 shot_id 的继承**：
   - 现有逻辑在 `taskService.js` 或 `batchScheduler.js` 中
   - 需找到 output 任务创建的位置，确保从 draft 复制 `shot_id`

3. **删除 shot 的安全处理**：
   ```javascript
   // shots.js DELETE 路由中
   const deleteShot = db.transaction((shotId) => {
     db.prepare('UPDATE tasks SET shot_id = NULL WHERE shot_id = ?').run(shotId);
     db.prepare('DELETE FROM shots WHERE id = ?').run(shotId);
   });
   ```

4. **删除 episode 的级联处理**：
   SQLite 的 `ON DELETE CASCADE` 会自动删除 episode 下的 shots。但 tasks.shot_id 不受外键保护（ALTER TABLE 添加的），需要应用层处理：
   ```javascript
   const deleteEpisode = db.transaction((episodeId) => {
     // 获取该集下所有镜头 ID
     const shots = db.prepare('SELECT id FROM shots WHERE episode_id = ?').all(episodeId);
     for (const shot of shots) {
       db.prepare('UPDATE tasks SET shot_id = NULL WHERE shot_id = ?').run(shot.id);
     }
     // CASCADE 会自动删除 shots
     db.prepare('DELETE FROM episodes WHERE id = ?').run(episodeId);
   });
   ```

5. **中间件提取是前置任务**：
   路由拆分前必须先将 `authenticate` 和 `requireAdmin` 提取到独立文件。这是一个独立的小 PR，建议先做。
