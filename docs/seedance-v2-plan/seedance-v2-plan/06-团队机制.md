# M6: 团队机制（多团队支持）

## 需求描述

支持多个团队，每个团队内成员共享项目、集、镜头和视频。未来不同团队之间数据隔离。

> 优先级 P2，当前阶段可以先按"所有用户同属一个默认团队"实现，为多团队预留数据结构。

## 数据库设计

### 新建 teams 表

```sql
CREATE TABLE teams (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT NOT NULL,                  -- 团队名称
  description TEXT DEFAULT NULL,
  created_by INTEGER NOT NULL REFERENCES users(id),
  created_at DATETIME DEFAULT (datetime('now')),
  updated_at DATETIME DEFAULT (datetime('now'))
);
```

### 新建 team_members 表

```sql
CREATE TABLE team_members (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  team_id INTEGER NOT NULL REFERENCES teams(id) ON DELETE CASCADE,
  user_id INTEGER NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  role TEXT NOT NULL DEFAULT 'member',  -- 'owner' | 'admin' | 'member'
  joined_at DATETIME DEFAULT (datetime('now')),
  UNIQUE(team_id, user_id)
);
```

### 修改 projects 表

```sql
ALTER TABLE projects ADD COLUMN team_id INTEGER DEFAULT NULL REFERENCES teams(id);
```

项目归属于团队，团队内所有成员可见。

## 数据隔离规则

| 数据 | 可见范围 |
|------|---------|
| 项目/集/镜头 | 同一团队所有成员 |
| 生成任务/视频 | 同一团队所有成员（可以看到谁生成的） |
| 即梦账号 | 全局（管理员管理，所有团队共用） |
| 个人设置 | 仅自己 |

## 阶段实施

### 第一阶段（当前）

- 创建数据表但暂不做 UI
- 系统自动创建一个"默认团队"
- 所有新注册用户自动加入默认团队
- 项目自动归属默认团队
- 相当于所有人在同一个团队，行为与当前一致

### 第二阶段（未来）

- 团队管理 UI：创建团队、邀请成员
- 项目归属团队选择
- 团队间数据隔离
- 团队内角色（团队管理员 vs 成员）

## API 设计（第二阶段）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/teams` | 获取我所在的团队列表 |
| POST | `/api/teams` | 创建团队 |
| GET | `/api/teams/:id/members` | 获取团队成员 |
| POST | `/api/teams/:id/members` | 邀请成员 |
| DELETE | `/api/teams/:id/members/:userId` | 移除成员 |
