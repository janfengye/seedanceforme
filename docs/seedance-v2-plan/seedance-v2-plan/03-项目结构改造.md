# M3: 项目结构改造（项目 → 集 → 镜头）

## 需求描述

将当前扁平的"项目"概念改造为三级结构：**项目 → 集（Episode）→ 镜头（Shot）**。

## 数据模型

### 层级关系

```
项目（Project）
  ├── 代号: SN
  ├── 名称: 十年
  │
  ├── 集 1（Episode）
  │     ├── 镜头 1（Shot）→ 生成任务 → 视频版本
  │     ├── 镜头 2（Shot）→ 生成任务 → 视频版本
  │     └── 镜头 3（Shot）
  │
  └── 集 2（Episode）
        ├── 镜头 1（Shot）
        └── 镜头 2（Shot）
```

### 数据库设计

#### 修改 projects 表

```sql
ALTER TABLE projects ADD COLUMN code TEXT DEFAULT NULL;
-- code: 项目代号，如 SN，用于文件命名前缀

-- 第一阶段：全局唯一索引（所有人同属一个默认团队）
-- 第二阶段（M6 多团队）：改为 ON projects(team_id, code) WHERE code IS NOT NULL
CREATE UNIQUE INDEX IF NOT EXISTS idx_projects_code
  ON projects(code) WHERE code IS NOT NULL;
```

#### 新建 episodes 表

```sql
CREATE TABLE episodes (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  project_id INTEGER NOT NULL REFERENCES projects(id) ON DELETE CASCADE,
  episode_number INTEGER NOT NULL,        -- 集数：1, 2, 3...
  title TEXT DEFAULT NULL,                -- 可选标题，如"序章"
  description TEXT DEFAULT NULL,
  created_at DATETIME DEFAULT (datetime('now')),
  updated_at DATETIME DEFAULT (datetime('now')),
  UNIQUE(project_id, episode_number)
);
```

#### 新建 shots 表

```sql
CREATE TABLE shots (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  episode_id INTEGER NOT NULL REFERENCES episodes(id) ON DELETE CASCADE,
  shot_number INTEGER NOT NULL,           -- 镜头号：1, 2, 3...
  description TEXT DEFAULT NULL,          -- 镜头描述/备注
  prompt TEXT DEFAULT NULL,               -- 默认提示词（可选）
  reference_image_url TEXT DEFAULT NULL,  -- 参考图（可选）
  preferred_model TEXT DEFAULT NULL,     -- 推荐模型（如 dreamina_seedance_40_pro）
  status TEXT DEFAULT 'active' CHECK (status IN ('active', 'archived')),  -- 镜头状态
  created_at DATETIME DEFAULT (datetime('now')),
  updated_at DATETIME DEFAULT (datetime('now')),
  UNIQUE(episode_id, shot_number)
);
```

#### 修改 tasks 表

```sql
-- 注意：SQLite 的 ALTER TABLE ADD COLUMN 添加的 REFERENCES 不会被实际检查，
-- 因此不写外键声明，改由应用层保证引用完整性。
-- 删除镜头时必须先执行：UPDATE tasks SET shot_id = NULL WHERE shot_id = ?
ALTER TABLE tasks ADD COLUMN shot_id INTEGER DEFAULT NULL;
```

关联生成任务到具体镜头。

> **注意**：SQLite 的 `ALTER TABLE ADD COLUMN` 添加的外键约束不会被实际检查。因此不写 `REFERENCES shots(id)`，改由应用层保证：删除镜头时先执行 `UPDATE tasks SET shot_id = NULL WHERE shot_id = ?`，再删除 shot 记录。

## 权限模型

> **核心原则：管理员管理结构，普通用户使用结构。**

| 操作 | 管理员 | 普通用户 |
|------|--------|---------|
| 创建项目 | ✅ | ❌ |
| 编辑项目（含设置代号） | ✅ | ❌ |
| 删除项目 | ✅ | ❌ |
| 创建/编辑/删除集 | ✅ | ❌ |
| 创建/编辑/删除镜头 | ✅ | ❌ |
| 查看项目/集/镜头 | ✅ | ✅ |
| 生成视频并关联镜头 | ✅ | ✅ |
| 查看镜头版本列表 | ✅ | ✅ |

**设计理由**：项目/集/镜头是团队共享的内容结构，由管理员统一规划。普通用户只负责在已有结构下生成视频。这与传统影视制作中"制片人规划镜头列表、艺术家在指定镜头下提交作品"的流程一致（参考 Kitsu 的 Production Manager / Artist 角色分工）。

## API 设计

### 项目

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| PUT | `/api/projects/:id` | 更新项目（新增 code 字段） | 管理员 |

### 集

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| GET | `/api/projects/:projectId/episodes` | 获取项目下所有集 | 登录用户 |
| POST | `/api/projects/:projectId/episodes` | 创建新集 | 管理员 |
| PUT | `/api/episodes/:id` | 更新集信息 | 管理员 |
| DELETE | `/api/episodes/:id` | 删除集（级联删除镜头） | 管理员 |

### 镜头

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| GET | `/api/episodes/:episodeId/shots` | 获取集下所有镜头 | 登录用户 |
| POST | `/api/episodes/:episodeId/shots` | 创建新镜头 | 管理员 |
| PUT | `/api/shots/:id` | 更新镜头信息 | 管理员 |
| DELETE | `/api/shots/:id` | 删除镜头 | 管理员 |

## 前端页面改动

### 修改：项目管理页

项目创建/编辑时增加「项目代号」字段：

```
项目名称：十年
项目代号：SN  （2-10字符，英文大写+数字）
```

展示集和镜头的树形结构：

```

│  十年（SN）                                      │

│  ┌───────┬───────┬───────┬──────────────────┐   │

│  ├───────┼───────┼───────┼──────────────────┤   │
│  │ SN-1-1│ 开场   │ 3     │ [生成] [查看]    │   │
│  │ SN-1-2│ 对话   │ 1     │ [生成] [查看]    │   │
│  │ SN-1-3│ 结尾   │ 0     │ [生成]           │   │
│  └───────┴───────┴───────┴──────────────────┘   │
│                                                 │
│  第 2 集                          [+ 添加镜头]   │
│  ┌───────┬───────┬───────┬──────────────────┐   │

│                                                 │
│  [+ 添加新集]                                    │
│                                                 │
```

### 修改：生成页面

生成视频时增加选择流程：选择项目 → 选择集 → 选择镜头

> **注意**：普通用户只能从已有结构中选择，不能新建项目/集/镜头。"+ 新建镜头"入口仅管理员可见。
