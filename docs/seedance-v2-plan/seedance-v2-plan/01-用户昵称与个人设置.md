# M1: 用户昵称与个人设置页

## 需求描述

每个用户可以设置自己的**昵称（nickname）**，昵称将作为视频文件命名中的用户简写标识。

## 当前现状

- `users` 表有 `username`、`email`、`role` 等字段
- 无个人设置页面
- 无昵称字段

## 数据库改动

### users 表新增字段

```sql
ALTER TABLE users ADD COLUMN nickname TEXT DEFAULT NULL;
```

- `nickname`：用户昵称/简写，如 `XXX`、`LM`、`ZH`
- 用于文件命名，建议限制：2-10 字符，仅英文字母+数字，不允许特殊字符
- 唯一性约束：同一团队内不可重复

## API 设计

### GET /api/user/profile

获取当前用户个人信息。

**响应：**
```json
{
  "id": 1,
  "username": "zhangsan",
  "email": "zhangsan@example.com",
  "nickname": "ZS",
  "role": "user",
  "created_at": "2026-04-10T10:00:00Z"
}
```

### PUT /api/user/profile

更新个人信息。

**请求：**
```json
{
  "nickname": "ZS"
}
```

**校验规则：**
- 长度：2-10 字符
- 格式：仅允许英文字母和数字（`/^[A-Za-z0-9]+$/`）
- 唯一性：不可与其他用户重复
- 首次进入个人设置页时，如果未设置昵称，提示必须设置
