# M2: 即梦账号权限隔离

## 需求描述

- **管理员**：可以完整管理即梦 SessionID 账号（增删改查、测试连接）
- **普通成员**：设置页仅显示"可用账号 X 个"及可用状态摘要，不暴露任何 SessionID

## 当前现状

- 设置页（`/settings`）所有用户都能看到完整的即梦账号列表和 SessionID
- `jimeng_session_accounts` 表存储所有即梦账号
- 后端 API 无角色区分，所有接口返回完整数据

## 后端改动

### API 权限控制

#### GET /api/settings/session-accounts（已有）

**管理员响应（不变）：** 完整账号列表，含 SessionID

**普通成员响应（新增逻辑）：**
```json
{
  "summary": {
    "total": 5,
    "available": 3,
    "unavailable": 2
  }
}
```

不返回具体账号列表和 SessionID。

#### POST/PUT/DELETE /api/settings/session-accounts/*

- 仅管理员可调用
- 普通成员调用返回 `403 Forbidden`

### 实现方式

在路由中间件中检查 `req.user.role`：

```javascript
function adminOnly(req, res, next) {
  if (req.user.role !== 'admin') {
    return res.status(403).json({ error: '仅管理员可执行此操作' });
  }
  next();
}
```
