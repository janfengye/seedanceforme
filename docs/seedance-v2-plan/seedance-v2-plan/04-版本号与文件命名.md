# M4: 版本号与文件命名

## 需求描述

视频生成后，下载文件自动按规则命名：

```
{项目代号}-{集数}-{镜头号}-{版本号}-{用户昵称}.mp4
```

示例：`SN-1-1-A-XXX.mp4`

## 版本号规则

- 版本号为大写字母：A, B, C, D, ..., Z, AA, AB, ...
- **按用户各自递增**：每个用户在同一镜头下独立计算版本号
- 示例：
  - 用户 XXX 在镜头 SN-1-1 下第1次生成 → `SN-1-1-A-XXX`
  - 用户 YYY 在镜头 SN-1-1 下第1次生成 → `SN-1-1-A-YYY`
  - 用户 XXX 在镜头 SN-1-1 下第2次生成 → `SN-1-1-B-XXX`

## 数据库改动

### tasks 表新增字段

```sql
ALTER TABLE tasks ADD COLUMN version_label TEXT DEFAULT NULL;
-- 存储计算后的版本标签，如 'A', 'B', 'C'
```

### 版本号计算逻辑（后端）

当任务关联到镜头（`shot_id`）并生成成功时，自动计算版本号：

```javascript
function getNextVersionLabel(shotId, userId) {
  const count = db.prepare(
    'SELECT COUNT(*) as cnt FROM tasks WHERE shot_id = ? AND user_id = ? AND version_label IS NOT NULL'
  ).get(shotId, userId).cnt;
  return numberToLabel(count); // 0→A, 1→B, 2→C, ...25→Z, 26→AA
}

function numberToLabel(n) {
  let label = '';
  do {
    label = String.fromCharCode(65 + (n % 26)) + label;
    n = Math.floor(n / 26) - 1;
  } while (n >= 0);
  return label;
}
```

## 文件命名逻辑

### 生成完整文件名

```javascript
function buildFilename(task) {
  // task 需要关联查询 shot → episode → project 以及 user
  const { project_code, episode_number, shot_number, version_label, nickname } = task;
  return `${project_code}-${episode_number}-${shot_number}-${version_label}-${nickname}.mp4`;
}
```

### 应用场景

| 场景 | 说明 |
|------|------|
| 下载管理页下载 | 使用规范文件名 |
| 批量下载 | 使用规范文件名 |
| 从即梦同步的视频 | 如果能匹配到镜头则命名，否则使用默认名 |

### 未关联镜头的任务

如果用户直接生成视频但未选择项目/集/镜头，文件名降级为：

```
video_{taskId}_{timestamp}.mp4
```
